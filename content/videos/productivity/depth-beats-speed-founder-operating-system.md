## Section 1: Depth Compounds Faster Than Speed — 3 Minutes
Host:

"Speed feels productive. Depth is productive. Depth is when you stick with the important work long enough for compounding to kick in—skills sharpen, systems stabilize, customers trust you.

Most teams ping‑pong between tasks, meetings, and Slacks. The way out? Protect depth with 90‑minute focus blocks. No notifications, no context switches, one outcome. That one block a day, five days a week, becomes your unfair advantage."

Key ideas:

1) Depth > speed for compounding
- Speed spreads attention thin; depth stacks gains in one direction.
- Compounding comes from continuity, not bursts.

2) The 90‑minute rule
- Choose one needle‑moving outcome per block.
- Prep a tiny checklist before you start; no hunting during the block.

3) Guardrails that make depth automatic
- Calendar the block first; meetings fit around it.
- Phone on airplane mode; desktop Do Not Disturb.
- Work in a “clean room” workspace (just the files and tools you need).

Example:

- Instead of “work on product,” the block is “ship checkout error telemetry to dashboard.” Clear, shippable, measurable.

## Section 2: Decision Speed — Slow for One‑Way, Fast for Two‑Way — 3 Minutes
Host:

"Move slow on one‑way doors. Move fast on two‑way doors. This is how you keep quality high without killing momentum."

Definitions:

- One‑way decisions: hard to reverse (pricing model change, database vendor, brand rename). Be slow: gather data, run small tests, invite dissent.
- Two‑way decisions: easy to reverse (button copy, feature flag default, ad headline). Be fast: ship now, measure, revert if needed.

Simple playbook:

1) Label the door
- Ask: if we’re wrong, how hard is the rollback?

2) Match the process to the door
- One‑way: write a 1‑pager, list risks, define kill switches, set a deliberate decision date.
- Two‑way: set a 24–48h deadline, ship behind a flag, decide by metrics not opinions.

3) Make the smallest bet first
- One‑way: simulate or pilot with 5% of traffic/users.
- Two‑way: A/B test or shadow‑ship with logging only.

Example:

- One‑way: Moving to a new billing provider → run a shadow pipeline for a week, verify parity, then cut over.
- Two‑way: Try a new onboarding headline → ship to 20% today, keep the winner by Friday.

(On screen: a “Door A: One‑Way” sign vs “Door B: Two‑Way” sign with arrows)

## Section 3: Daily System — 90‑Min Blocks, Delegate 30%, 1% Chain — 3 Minutes
Host:

"Momentum becomes mastery when your daily system is boring and repeatable. Three rules: 90‑minute blocks, delegate the 30%, and don’t break the chain on 1% improvements."

1) Protect one 90‑minute block (depth)
- Put it on the calendar before anything else.
- End each block with a 2‑minute log: What shipped? What’s next?

2) Delegate the 30% others can do 80% as well
- If someone can do it 80% as well with a checklist, delegate it.
- Ship the process, not just the task: provide examples, definition of done, and a review cadence.
- Keep the highest‑leverage 70%: vision, architecture, key relationships, decisions.

3) 1% daily improvements (don’t break the chain)
- Every day, upgrade something tiny: a test, a script, a query, a doc.
- Track a visible chain (calendar X’s, Git streak, habit app). The chain is the game.
- One rule: never miss twice.

Daily cadence (sample):

- 08:30–09:00: Triage and delegate (turn Slack/Email into tasks, assign owners).
- 09:00–10:30: 90‑minute focus block (depth outcome).
- 10:30–10:40: Log shipping + pick tomorrow’s block.
- 10:40–11:00: 1% improvement (small refactor, monitor, doc, automation).

Example:

- Monday chain: add alerting on 500s → Tuesday: tighten retry policy → Wednesday: index slow query → Thursday: doc failover → Friday: auto‑rollbacks on health check. Tiny steps, big reliability.
