## Hook
If you run long AI tasks in chat, this OpenClaw update matters immediately: you can now check progress with a built-in `/tasks` board.

## 3 Key Points
1. **What happened:** OpenClaw 2026.4.1 introduced `/tasks`, a chat-native background task board that shows recent task details in-session.
2. **Why it matters now:** Teams using agent workflows often lose track of what is still running, failed, or completed, and this feature reduces that coordination overhead.
3. **What to do next:** Update to 2026.4.1 or newer, test `/tasks` in one active project channel, and define who owns follow-up when tasks fail.

## Full Script (60–90 second talking-head)
Here’s a useful quality-of-life update if you use AI agents in real workflows.

OpenClaw 2026.4.1 added a `/tasks` command that gives you a chat-native board for background runs.

Why this matters is simple: once you have multiple agent tasks running, people stop knowing what is active versus done versus stuck. That creates delay and duplicated effort.

According to the release notes, `/tasks` surfaces recent task details for the current session. The same release also includes task-related fixes around stale status visibility and gateway sweep behavior, which suggests the team is tightening reliability around background execution, not just adding a shiny command.

Practical move: roll this out in one team workflow first. Use `/tasks` during daily check-ins so everyone sees the same task state before asking for re-runs.

That gives you cleaner handoffs and fewer “is this still running?” messages.
