## Full Script (60-90 seconds)
Quick update for engineering teams running GitHub Copilot across multiple tools.

GitHub’s official changelog confirms Claude Opus 4.6 is now generally available in Copilot, including GitHub.com, mobile, VS Code, Visual Studio, JetBrains IDEs, Xcode, and Eclipse. GitHub’s Copilot model documentation also confirms model availability varies by plan and surface, so it’s worth validating access in your environment before rollout.

Why this matters: most teams don’t code in one editor anymore. When model access differs by tool, output quality and review effort can become inconsistent. A broader rollout lets you compare one model under real team conditions.

Here’s a practical play for today: pick three tasks from your current sprint—one bug fix, one refactor, one test-generation task. Run your current default model and Opus 4.6 against the same tasks. Measure time to first acceptable output, number of manual edits before merge, and reviewer correction comments.

If Opus 4.6 wins clearly on your own repo metrics, promote it for those task types. If not, keep it as an escalation model for harder tickets.

Don’t switch because it’s new. Switch because your team’s data says it improves delivery quality and speed.
