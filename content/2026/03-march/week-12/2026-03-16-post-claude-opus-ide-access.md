## Full Script (60-90 seconds)
Quick heads-up for developers using GitHub Copilot.

GitHub has announced Claude Opus 4.6 availability across major environments, including VS Code, Visual Studio, JetBrains IDEs, Xcode, Eclipse, plus GitHub.com and mobile.

This matters because model selection is now a workflow decision, not just a preference. If your team is split across tools, using one high-capability model across all of them can reduce context drift, make reviews more predictable, and improve prompt reuse.

Here’s the practical way to use this: don’t switch blindly. Run a small internal benchmark today. Pick three real tasks from your backlog—one bug fix, one refactor, and one test-writing task. Compare output quality, edit distance, and time-to-merge against your current default model.

If the win is clear, standardize. If not, keep Opus 4.6 as a targeted option for harder tasks.

Either way, treat model choice like a measurable engineering decision.
