## Full Script (60-90 seconds)
Quick update for engineering teams using GitHub Copilot.

GitHub announced Claude Opus 4.6 is now available in Copilot across major environments: GitHub.com, mobile, VS Code, Visual Studio, JetBrains, Xcode, and Eclipse.

Why this matters: model choice now affects team throughput. If everyone is using different models in different tools, prompt behavior and review quality become inconsistent.

The practical play is not “switch everyone today.” Do a controlled rollout.

Pick three recurring task types from your backlog: one bug fix, one medium refactor, and one test-generation task. For one week, run those on Opus 4.6 and track two numbers: time-to-merge and reviewer-requested changes.

If you see a clear quality or speed improvement, then expand usage policy. If results are mixed, keep it as a targeted model for harder tasks only.

Treat model selection like any other engineering optimization: measured, not emotional.
