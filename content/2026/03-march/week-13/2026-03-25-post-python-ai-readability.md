## Hook
If you think Python is “falling off” because TypeScript hit #1 on GitHub, you’re reading the chart wrong.

## 3 Key Points
1. **What changed:** GitHub’s 2025 Octoverse says TypeScript became the most used language on GitHub in August 2025.
2. **What didn’t change:** In GitHub’s interview, Guido van Rossum points out Python still grew strongly in 2025 and remains central for AI and data work.
3. **Practical move for developers:** Use TypeScript where typed app surfaces matter, but keep Python as your fast lane for AI prototyping, data workflows, and model experimentation.

## Full Script (60–90 seconds)
Quick reality check for devs saying Python is done.

Yes—TypeScript became the most used language on GitHub in 2025. That’s real, and it makes sense with modern app stacks and agent-assisted coding.

But that doesn’t mean Python is declining where it matters most for AI builders.

GitHub’s own coverage of Guido van Rossum highlights that Python still saw strong growth and continues to be the default language across AI, science, and education workflows.

So the smart takeaway is not “pick one language forever.”

Use TypeScript for production-facing app layers where type safety helps teams ship and review faster.

Use Python for rapid model iteration, evaluation pipelines, data prep, and experimentation where ecosystem speed still dominates.

In other words: don’t do language tribalism—do workload matching.

If your team is building AI features in 2026, the win is a bilingual stack with clear boundaries, not a winner-take-all argument.
