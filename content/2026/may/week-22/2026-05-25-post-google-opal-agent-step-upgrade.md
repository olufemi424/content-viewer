## Hook
Google just changed Opal from a prompt toy into a real agent builder.

## Why this matters
If you are a solo builder, this cuts setup time: instead of wiring every model call manually, you give Opal a goal and it picks tools, asks follow-up questions, and routes work for you. That means faster prototypes and less glue logic.

## Mechanism
Google Labs added a new **agent step** in Opal (announced Feb 24, 2026). In that step, Opal can:
- choose tools/models (like Web Search or Veo) based on your objective
- keep memory across sessions
- branch with dynamic routing logic
- ask users clarifying questions through interactive chat

## Proof/use case
Google's own examples show the shift:
- story workflows moved from fixed page/question templates to a Visual Storyteller flow where the agent decides what details to request next
- Room Styler now behaves like an iterative design partner, taking feedback and refining outputs instead of one-shot generation
For creator/operators, this is the practical signal: you can build mini-apps that adapt per user input without hand-coding orchestration.
