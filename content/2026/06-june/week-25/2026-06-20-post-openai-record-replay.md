## Hook
OpenAI just shipped a feature that could save creators and operators from repeating the same browser-and-app chores every week: show Codex the workflow once, and it turns that demo into a reusable skill.

## Why this matters
Most people still automate the hard way. They either write a script, build a plugin, or keep re-prompting the same checklist every time a recurring task comes back. That is fine for engineering-heavy teams, but it is friction for solo builders, creators, and operators who just want the workflow captured. Record & Replay matters because it moves automation one step earlier: from "write the system" to "demonstrate the routine." The practical shift is that repeatable work like publishing a video, filing an expense, creating a correctly configured issue, or downloading a recurring report can start as a shown behavior instead of a coded integration.

## Mechanism
OpenAI added Record & Replay to the Codex app on June 18, 2026. It is a macOS feature that requires Computer Use to be available and enabled. The workflow is simple: start a recording from the Codex app, demonstrate the task on your Mac, stop recording, and Codex drafts an inspectable, editable skill from what it observed. OpenAI says that generated skill can later be reused with Computer Use, browser actions, installed plugins, or a mix of available tools. The key detail is that this is not a one-off macro recorder. The output is a reusable skill with guidance on when to use it, what inputs change, what steps matter, and how to verify the result.

## Proof/use case
The best proof is OpenAI's own launch framing because it names concrete workflows instead of vague productivity promises. The developer guide specifically lists filing an expense, booking a parking space, creating a correctly configured issue, publishing a video, and downloading a recurring report as example use cases. The Codex changelog confirms the launch date and scope, while the ChatGPT Business release notes add the team angle: stable workflows that are easier to show than describe can be reused with Codex across Computer Use, browser actions, and plugins. For Olufemi's audience, the practical use case is obvious: if a creator has a weekly publishing routine or an operator has a recurring reporting flow, they can record the clean version once, refine the resulting skill, and stop rebuilding the process from scratch every time.
