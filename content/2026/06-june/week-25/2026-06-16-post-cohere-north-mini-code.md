## Hook
Cohere just shipped an open coding model with a very specific promise: real agent workflows, without needing a giant hosted stack.

## Why this matters
Most builder talk around coding AI still sounds like benchmark theater. North Mini Code is more practical than that. Cohere says it is an open-weight model built for code generation, terminal tasks, code reviews, and sub-agent style workflows, while staying small enough to run locally or on your own infrastructure. For solo builders and operators, that matters because the next bottleneck is not getting one smart answer. It is getting repeatable coding workflows you can actually control, audit, and afford to run more than once.

## Mechanism
According to Cohere's launch materials and docs, North Mini Code is a 30B total / 3B active mixture-of-experts model released under Apache 2.0 with a 256K context window and 64K max output. Cohere positions it for agentic coding rather than plain autocomplete, including software-engineering tasks, terminal use, code review, and orchestrating sub-agents. The core pitch is efficiency plus control: open weights on Hugging Face, API access through Cohere, and a small enough active footprint to make local or on-prem deployment realistic for teams that do not want all coding context leaving their stack.

## Proof/use case
The clearest use case is a solo builder or small team running repo work in a local or governed environment: inspect a codebase, execute terminal steps, review a patch, then loop again without handing everything to a closed hosted agent. Cohere cites competitive coding-benchmark performance and says it reached up to 2.8x higher output throughput than Devstral Small 2 in internal testing. Artificial Analysis also listed North Mini Code at 33.4 on its Coding Index, which at least supports the bigger claim that this is not just a cheap model, but one built to survive real coding loops better than generic small models.
