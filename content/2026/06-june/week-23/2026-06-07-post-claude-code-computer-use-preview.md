## Hook
If you use AI to build software, Claude Code just crossed from terminal helper to desktop operator.

## Why this matters
Most coding agents stop at files and terminal commands. Anthropic's new computer use mode matters because Claude can now click through desktop apps, browsers, simulators, and local tools when no connector exists. For solo builders, that turns some annoying handoff work, like checking a UI, filling a dashboard, or testing an internal tool, into something the agent can help execute instead of just describe.

## Mechanism
Anthropic says computer use is now available in Cowork and Claude Code inside the Claude Desktop app on macOS and Windows, in research preview for Pro and Max plans. The system prefers direct tools first, then browser automation, then full screen interaction. Claude asks for per-app permission, can be stopped at any point, and Anthropic explicitly warns there is no sandbox between Claude and your applications, which is why they recommend starting with simple tasks and avoiding sensitive apps.

## Proof/use case
Anthropic's own examples are practical: open a phone simulator and find UX issues, navigate tools that do not have connectors, and compile reports from local files plus connected apps. The Claude Code desktop docs also tie this to previewing your app, watching Claude verify changes, and running several sessions in parallel. That is the useful angle: not "AI does everything," but "your agent can now touch the local software layer that used to break the workflow."
