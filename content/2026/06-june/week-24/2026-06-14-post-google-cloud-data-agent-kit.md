## Hook
Google Cloud just made a big bet on where coding agents go next: not broader prompts, but specialized skills.

## Why this matters
Google Cloud's new Data Agent Kit packages data engineering and data science workflows into installable skills, tools, and plugins that work inside surfaces people already use, including Codex, Claude Code, Gemini CLI, VS Code, and Antigravity CLI. For solo builders and operators, that matters because a lot of AI tooling still breaks the moment the work gets domain-specific. Instead of pasting schemas, hopping across consoles, and improvising every query or pipeline step, the pitch here is that the workflow itself becomes reusable.

## Mechanism
According to Google's launch post, Data Agent Kit combines three layers: predefined agentic skills, MCP tools that connect to Google Cloud data systems like BigQuery and Cloud Storage, and native integrations for IDE or CLI workflows. Google says the kit can route work across tasks like query optimization, ML best practices, data validation, data drift checks, and pipeline building. The official docs frame it as a way to manage data assets, run queries, and deploy data pipelines from your preferred workspace, while the starter pack repo shows install paths for Codex, Claude Code, and Antigravity-based workflows.

## Proof/use case
The best use case is not "AI does data now." It is a builder who already lives in a coding agent and wants to go from raw files to a usable pipeline without stitching the whole flow together manually. Google's example walks through moving transaction logs from Cloud Storage into BigQuery and dbt-style cleanup steps, then orchestrating the pipeline from the same interface. The starter pack repo also shows concrete workflow examples like creating Spark notebooks, transformation pipelines, and scheduled jobs, which makes this feel more like an installable operating layer than a vague demo.
