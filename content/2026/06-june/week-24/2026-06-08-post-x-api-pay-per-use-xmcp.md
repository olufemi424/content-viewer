## Hook
If you wanted to build on X data but could not justify a fixed monthly API bill, that just changed.

## Why this matters
X's API is now pay-per-use instead of subscription-first, and X also ships an official MCP server called XMCP. For solo builders, that means you can prototype search, posting, or agent workflows without committing to a big upfront plan. The catch is that "cheaper to start" does not mean "cheap at scale," especially if your workflow posts links or reads lots of data.

## Mechanism
X's docs say you pre-purchase credits and pay per resource or request instead of paying for a fixed tier. The same docs show official Python and TypeScript SDKs, plus XMCP, which turns X API endpoints into MCP tools for AI clients. In practice, that means a builder can wire X into an agent workflow faster, but still has to watch per-call costs and plan limits closely.

## Proof/use case
The useful case is simple: if you are building a creator dashboard, trend monitor, or posting assistant, you can now test real X workflows in smaller increments. X's docs position pay-per-use for hobbyists, prototyping, and early-stage integrations. TechCrunch also reported that X raised the price for posting links to $0.20 each in April 2026, which is why the winning move is prototyping narrow workflows first instead of assuming the economics stay friendly forever.
